现在讨论一个新的功能，suggestions。

该功能根据当前项目的情况建议应该安装哪些LSP/linter/formatter等工具。

## 核心逻辑

### 关于如何检测建议的工具，有两个思路

1. 在omp启动后其首页界面本身就会显示lsp servers，会显示已经安装的工具。这说明omp自身可能会有一定的扫描功能，根据本地项目内容检查是否需要什么LSP

2. 本插件自身根据本地项目内容进行扫描后建议

这两个思路都需要进行调研反馈，讨论哪个更合适。需要结合功能完整程度和操作所需时间综合考虑。

## UI 设计

- 在进入 /mason 面板后，tab行增加一个suggested入口，排在第二个，list之后。该入口应展示对于本地项目建议安装的工具

- 根据工具是否已安装来显示不同的状态：

  - 绿色对钩表示已经安装了，未安装的无标识

  - 光标移动到每个条目上面时提示快捷键，且应根据安装状态来对应显示合理的操作。

- UI设计请遵循 @docs/UI.md。
